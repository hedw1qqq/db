ЛР5: параллельное выполнение и уровни изоляции
Подключение: postgresql://admin:admin@localhost:5434/lab4

[Эксперимент 1] Блокировка одной строки
Сценарий: T1 обновляет объект estate.id=201 и держит транзакцию открытой 4 секунды.
T2 пытается обновить ту же строку, пока T1 еще не завершилась.
Ожидание T2: 3.959 с
Итоговая цена объекта: 4800.00
Вывод: блокировка строки действительно возникла, вторая транзакция ждала освобождения ресурса.

[Эксперимент 2] Сравнение уровней изоляции
Проверяем прикладное правило: для объекта 201 в августе должна сохраниться только одна новая бронь.
Обе транзакции сначала читают число броней, затем при необходимости пытаются вставить свою запись.

READ COMMITTED
--------------
- T1: COMMIT, увидела 0 подходящих броней
- T2: COMMIT, увидела 0 подходящих броней
- Итоговое число августовских броней: 2
- guest_id в итоговых бронях: [102, 103]
- Результат проверки: FAIL
- Вывод: Бизнес-правило нарушено: обе транзакции смогли добавить бронь независимо друг от друга.

REPEATABLE READ
---------------
- T1: COMMIT, увидела 0 подходящих броней
- T2: COMMIT, увидела 0 подходящих броней
- Итоговое число августовских броней: 2
- guest_id в итоговых бронях: [102, 103]
- Результат проверки: FAIL
- Вывод: Бизнес-правило нарушено: обе транзакции смогли добавить бронь независимо друг от друга.

SERIALIZABLE
------------

- Результат проверки: PASS
- Вывод: Бизнес-правило сохранено: в августе осталась только одна новая бронь.
